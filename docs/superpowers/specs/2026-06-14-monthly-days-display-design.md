# 「今月◯日」を主役にする — 設計書

作成日: 2026-06-14

## 背景・目的

現状アプリは「連続日数（れんぞく / streak）」をモチベーション指標として最も目立つ位置に表示している。
しかし仕事などで連続してトレーニングできない日が多く、連続が途切れるとモチベーションが下がる。

そこで **「1ヶ月のうち何日トレーニングしたか（今月のトレした日数）」** を主役の指標に切り替え、
連続日数は補助的に小さく残す。

## 方針（確定した決定事項）

1. **今月日数を主役、連続は小さく残す** — 連続日数を完全には消さず、控えめに併記する。
2. **表示はシンプルに「今月 12日」だけ** — 月間目標や達成率（◯/◯日）は設けない。
3. **連続バッジはそのまま残す** — バッジ画面（「3日れんぞく」等）は変更しない。

## スコープ

- 連続日数の計算ロジック `calcStreak()`（`main.go`）は**変更しない**。
- 連続バッジ（`badges.jsx`）は**変更しない**。
- API `done_days`（今月のトレした日数 = distinct な日付数）は**既に存在する**ため、
  バックエンドの計算追加は不要。`app.jsx` で各画面へ渡すだけ。

## 変更箇所

### ① カレンダー画面のバナー（`static/calendar.jsx`）

最も目立つ要素。主役を今月日数に差し替える。

- 現状: 大きく `{streak} DAYS` ＋ サブ「れんぞく きろくちゅう！」、左に炎スプライト
- 変更後:
  - 左にカレンダースプライト（`window.BADGE_CAL`）
  - 大きく `{doneDays} 日`
  - サブ「今月 トレした日数！」
  - さらに小さく炎アイコン＋「れんぞく {streak}日」を併記（控えめ）
- `doneDays` はこのコンポーネント内で `history` から既に算出済み。

### ② トップバー（`static/app.jsx`）

全画面共通の上部バー。

- 現状: 🔥 + `streak`、🪙 + `coins`
- 変更後: 📅(`window.BADGE_CAL`) + `今月日数` + 「日」（主役・オレンジ）、
  🔥 + `streak`（小さめ・`opacity:0.6`）、🪙 + `coins`
- `status.done_days` を `doneDays` として使用。

### ③ きろく画面ヘッダーのチップ（`static/homeInput.jsx`）

- 現状: 「れんぞく」チップ（🔥 + streak）、コインチップ
- 変更後: 「今月」チップ（📅 + 今月日数）を主役として先頭に追加、「れんぞく」チップは残す
- `HomeInput` に `doneDays` プロップを追加し、`app.jsx` から `status.done_days` を渡す。

## 変更しないもの

- カレンダー下部の「トレした日 {doneDays}日」サマリーカード（既存・そのまま）
- バッジ画面（連続バッジ含め全て）
- バックエンド `main.go`

## アイコンについて

カレンダーアイコンは `window.BADGE_CAL`（`app.jsx` で定義、ボトムナビでも使用済み）を再利用。
新規スプライト追加は不要。各 JSX はレンダリング時に `window.BADGE_CAL` を参照するため、
`app.jsx` が読み込み済みであれば問題ない（`index.html` の script 読み込み順で担保）。

## テスト・検証

- バックエンドにロジック変更がないため、既存の Go テスト（`main_test.go`）はそのまま通ること。
- フロントは静的JSXのため、ブラウザで各画面を開き、今月日数が正しく主役表示され、
  連続日数が控えめに残っていることを目視確認する。
